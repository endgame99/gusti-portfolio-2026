# PROJECT_BLUEPRINT.md
# Gustiansyah Portfolio Website 2026

## 0. Project Status

This project is a high-end personal portfolio / creative case-study website.

Important:
- The old portfolio PDF/material is NOT final content.
- The old material is only a basic placeholder / gambaran dasar.
- Final project content, visuals, case studies, images, and copy will change later.
- Build the website foundation first: layout, menu, UI/UX, section rhythm, motion, and flexible case-study system.
- Do not create a generic CV portfolio.

---

## 1. Website Vision

Create a premium, easy-to-consume, smooth portfolio website for a creative visual/design profile.

The website must feel like:

- AKQA as the dominant foundation:
  - premium agency structure
  - work-first navigation
  - editorial case-study layout
  - restrained, confident, not noisy

- Gemini About / Gemini Image Generation as the UI and reading experience:
  - clean white base
  - simple top navigation
  - short copy
  - smooth section flow
  - easy to scan
  - light, modern, polished

- ZCOOL as the visual-density reference only when needed:
  - long visual project detail pages
  - e-commerce/product/campaign visual showcase
  - dense image modules inside case studies
  - not for the default homepage layout

- Sofi.chat as contextual inspiration:
  - modern lookbook feeling
  - fashion/AI visual grid feeling
  - useful only when a project context fits that direction

Primary outcome:
A website that is clean, premium, flexible, and ready for future case-study content without looking like a template.

## 12. Visual Style System

### 12.1 Color

Base:
```txt
Background: #FFFFFF
Text: #111111
Muted Text: #5F6368 or similar
Border: #E5E5E5
Accent Blue: Gemini-like blue, use subtly
Surface: #F7F8FA
```

Rules:
- Blue accent is for highlights, links, active nav, and CTA.
- Do not overuse blue.
- Do not create a colorful theme without permission.
- Do not use random gradients.

### 12.2 Typography

Direction:
- clean sans-serif
- modern
- readable
- Gemini-like feeling
- premium spacing like AKQA

Suggested CSS stack:
```css
font-family: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

No paid font dependency in first build.

### 12.3 Spacing

Use generous spacing:
- desktop sections should breathe
- mobile should still be compact enough to read
- no cramped hero
- no tiny metadata

### 12.4 Images

Until final visuals are provided:
- use clean placeholder blocks
- do not fetch random images
- do not use unrelated stock images
- do not invent final project visuals
- image placeholders should be easy to replace later

## 13. Motion Behavior

Motion reference:
Gemini About smoothness.

Use:
- subtle fade-in
- text reveal
- image reveal
- smooth hover states
- card arrow movement
- gentle section entrance
- optional page transition later

Do not use:
- heavy cinematic animation in first build
- excessive parallax
- cursor gimmicks
- motion that hurts mobile performance
- animations that delay reading
- GSAP unless needed later

Initial motion library:
```txt
Motion / Framer Motion
```

GSAP:
```txt
Optional only for later polish.
Do not install in first build unless explicitly requested.
```

Accessibility:
- respect prefers-reduced-motion
- keep animations short
- no essential content hidden behind animation

Must test:

```txt

Mobile 390px
Tablet 768px–1024px
Large desktop
```

Mobile requirements:
- menu must appear and work
- hero text must not overflow
- CTA must be tappable
- work cards must stack cleanly
- no tiny text
- no horizontal scroll
- footer readable

Tablet requirements:
- navigation should switch to mobile menu if cramped
- work cards should form balanced rows
- no broken layout between 768px–1100px

## 19. Performance Requirements

- Keep first build lightweight.
- Avoid huge animation libraries.
- Use optimized image strategy.
- Do not load external random assets.
- Use system font first.
- Lazy-load below-fold media later.
